## Introduction to Transformers

### What is the Transformer?

The **Transformer** is a neural network architecture introduced in the 2017 paper "Attention is All You Need" by Vaswani et al. It revolutionized natural language processing by:
- Eliminating recurrence (no RNNs/LSTMs needed)
- Enabling parallel processing
- Capturing long-range dependencies effectively
- Scaling to billions of parameters

### Why Transformers?

**Problem with Previous Architectures (RNNs/LSTMs)**:
```
Sequential Processing:
word₁ → word₂ → word₃ → word₄ → word₅
  ↓       ↓       ↓       ↓       ↓
 h₁ →   h₂ →   h₃ →   h₄ →   h₅

Issues:
- Must process sequentially (slow)
- Gradient vanishing over long sequences
- Difficult to parallelize
- Limited context understanding
```

**Transformer Solution**:
```
Parallel Processing:
word₁, word₂, word₃, word₄, word₅
  ↓      ↓      ↓      ↓      ↓
All processed simultaneously with attention
  ↓      ↓      ↓      ↓      ↓
Each word "attends" to all other words

Benefits:
- Fully parallelizable (fast training)
- No gradient vanishing
- Better long-range dependencies
- Scales to very long sequences
```

### Key Innovation: Self-Attention

Instead of processing words sequentially, transformers use **self-attention** to process all words simultaneously while understanding their relationships.

```
Sentence: "The cat sat on the mat"

RNN: Process left-to-right, maintain hidden state
Transformer: All words processed together, each
            attending to all others
```

## Architecture Overview

### Full Transformer Architecture

```
┌─────────────────────────────────────────────────────────┐
│                   ENCODER                                │
├─────────────────────────────────────────────────────────┤
│  Input: "The cat sat on the mat"                        │
│      ↓                                                   │
│  [Token Embeddings + Positional Encoding]               │
│      ↓                                                   │
│  ┌─────────────────────────────────┐                    │
│  │  Multi-Head Self-Attention      │ ×N layers          │
│  │         ↓                        │ (typically 6-96)   │
│  │  Add & Layer Norm                │                    │
│  │         ↓                        │                    │
│  │  Feed-Forward Network            │                    │
│  │         ↓                        │                    │
│  │  Add & Layer Norm                │                    │
│  └─────────────────────────────────┘                    │
│      ↓                                                   │
│  Encoder Output (Context)                               │
└──────────────────┬──────────────────────────────────────┘
                   │
                   ↓ (Context passed to decoder)
┌─────────────────────────────────────────────────────────┐
│                   DECODER                                │
├─────────────────────────────────────────────────────────┤
│  Output: "Le chat s'est assis" (translation)            │
│      ↓                                                   │
│  [Token Embeddings + Positional Encoding]               │
│      ↓                                                   │
│  ┌─────────────────────────────────┐                    │
│  │  Masked Multi-Head Self-Attention│ ×N layers         │
│  │         ↓                        │                    │
│  │  Add & Layer Norm                │                    │
│  │         ↓                        │                    │
│  │  Cross-Attention (to encoder)   │                    │
│  │         ↓                        │                    │
│  │  Add & Layer Norm                │                    │
│  │         ↓                        │                    │
│  │  Feed-Forward Network            │                    │
│  │         ↓                        │                    │
│  │  Add & Layer Norm                │                    │
│  └─────────────────────────────────┘                    │
│      ↓                                                   │
│  Linear Layer + Softmax                                 │
│      ↓                                                   │
│  Output Probabilities                                   │
└─────────────────────────────────────────────────────────┘
```

### Core Components

1. **Input Embedding**: Convert tokens to vectors
2. **Positional Encoding**: Add position information
3. **Self-Attention**: Understand word relationships
4. **Multi-Head Attention**: Multiple attention perspectives
5. **Feed-Forward Network**: Process each position independently
6. **Layer Normalization**: Stabilize training
7. **Residual Connections**: Enable deep networks

## Input Processing

### 1. Tokenization

Breaking text into tokens:

```python
Text: "The quick brown fox"

Tokens: ["The", "quick", "brown", "fox"]
Token IDs: [284, 2068, 7586, 21831]
```

### 2. Token Embedding

Converting token IDs to dense vectors:

```
Token ID: 284 ("The")
     ↓
Embedding Layer (learned weights)
     ↓
Vector: [0.25, -0.45, 0.67, 0.12, ..., -0.33]
        (typically 512-12288 dimensions)
```

**Embedding Matrix**:
```
Vocabulary Size: 50,000 tokens
Embedding Dimension: 768

Embedding Matrix Shape: [50,000 × 768]
Each token → 768-dimensional vector
```

**Example**:
```
Sentence: "Cat sat"
Tokens: [cat, sat]
Token IDs: [5234, 8901]

Embeddings:
cat: [0.2, -0.5, 0.3, ..., 0.1]  (768 dims)
sat: [0.1, 0.3, -0.2, ..., 0.5]  (768 dims)
```

## Positional Encoding

### Why Positional Encoding?

Unlike RNNs, transformers process all tokens simultaneously. They need explicit position information:

```
Without position info:
"cat eats fish" ≈ "fish eats cat" (same embeddings, wrong meaning!)

With position info:
"cat eats fish" ≠ "fish eats cat" (different positional encodings)
```

### Sinusoidal Positional Encoding

**Formula**:
```
PE(pos, 2i) = sin(pos / 10000^(2i/d_model))
PE(pos, 2i+1) = cos(pos / 10000^(2i/d_model))

Where:
pos = position in sequence (0, 1, 2, ...)
i = dimension index
d_model = embedding dimension (e.g., 512)
```

### Visualization

```
Position 0: [sin(0/10000^0), cos(0/10000^0), sin(0/10000^(2/512)), ...]
Position 1: [sin(1/10000^0), cos(1/10000^0), sin(1/10000^(2/512)), ...]
Position 2: [sin(2/10000^0), cos(2/10000^0), sin(2/10000^(2/512)), ...]
...

Different patterns for each position
Allows model to learn relative positions
```

### Example

```python
# Simplified example
Sentence: "The cat sat"
Positions: [0, 1, 2]

Token Embeddings:
The: [0.5, 0.2, -0.3, 0.1]
cat: [0.1, -0.4, 0.2, 0.6]
sat: [-0.2, 0.3, 0.1, -0.1]

Positional Encodings:
Pos 0: [0.00, 1.00, 0.00, 1.00]
Pos 1: [0.84, 0.54, 0.01, 0.99]
Pos 2: [0.91, -0.42, 0.02, 0.99]

Final Input (Token Emb + Pos Enc):
The: [0.50, 1.20, -0.30, 1.10]
cat: [0.94, 0.14, 0.21, 1.59]
sat: [0.71, -0.12, 0.12, 0.89]
```

### Properties

- **Deterministic**: Same position → same encoding
- **Periodic**: Uses sine/cosine for patterns
- **Relative positions**: Can learn relative distances
- **Unbounded**: Works for any sequence length

## Self-Attention Mechanism

### Core Concept

Self-attention allows each word to "look at" every other word in the sentence to understand context.

```
Sentence: "The animal didn't cross the street because it was too tired"

Question: What does "it" refer to?

Self-Attention:
"it" attends to:
- "animal" (high attention score) ✓
- "street" (low attention score)
- "tired" (medium attention score)

Model learns "it" refers to "animal"
```

### Query, Key, Value (QKV)

Every token gets three vectors:
- **Query (Q)**: "What am I looking for?"
- **Key (K)**: "What do I contain?"
- **Value (V)**: "What information do I provide?"

```
Token Embedding
      ↓
  ┌───┴───┐───┐
  ↓       ↓   ↓
Query   Key  Value
(Q)     (K)  (V)

Created by multiplying embedding with learned weight matrices:
Q = Embedding × W_Q
K = Embedding × W_K
V = Embedding × W_V
```

### Attention Calculation Steps

**Step 1: Compute QKV**
```python
Example: "cat sat mat"

Embeddings (simplified, 4-dim):
cat: [1, 0, 1, 0]
sat: [0, 1, 0, 1]
mat: [1, 1, 0, 0]

After linear transformations:
Q_cat: [0.5, 0.2]
K_cat: [0.3, 0.4]
V_cat: [0.7, 0.1]
... (similar for sat, mat)
```

**Step 2: Calculate Attention Scores**
```
Score(Q_i, K_j) = Q_i · K_j / √d_k

Example: How much should "cat" attend to "sat"?
Score = (Q_cat · K_sat) / √d_k
      = (0.5×0.4 + 0.2×0.3) / √2
      = (0.20 + 0.06) / 1.41
      = 0.18
```

**Step 3: Apply Softmax**
```
Attention Weights = Softmax(Scores)

For "cat" attending to all words:
Scores:  [cat: 0.85, sat: 0.18, mat: 0.42]
Softmax: [cat: 0.49, sat: 0.24, mat: 0.27]
         (sum = 1.0)
```

**Step 4: Weighted Sum of Values**
```
Output = Σ (Attention_Weight_i × Value_i)

Output_cat = 0.49×V_cat + 0.24×V_sat + 0.27×V_mat
```

### Complete Formula

```
Attention(Q, K, V) = softmax(Q·K^T / √d_k) × V

Where:
Q: Query matrix
K: Key matrix (transposed)
V: Value matrix
d_k: Dimension of key vectors (for scaling)
√d_k: Prevents large values in softmax
```

### Worked Example

```
Sentence: "cat sat"

Step 1: Embeddings (2D simplified)
cat: [1, 0]
sat: [0, 1]

Step 2: Create Q, K, V (W matrices learned)
Q_cat = [1,0] × [[0.5],[0.2]] = [0.5]
Q_sat = [0,1] × [[0.5],[0.2]] = [0.2]

K_cat = [0.3], V_cat = [0.7]
K_sat = [0.4], V_sat = [0.9]

Step 3: Attention scores
Score(cat→cat) = 0.5 × 0.3 / √1 = 0.15
Score(cat→sat) = 0.5 × 0.4 / √1 = 0.20

Step 4: Softmax
Weights: [0.46, 0.54] (normalized)

Step 5: Weighted sum
Output_cat = 0.46×0.7 + 0.54×0.9 = 0.81
```

## Multi-Head Attention

### Why Multiple Heads?

Single attention focuses on one type of relationship. Multiple heads can capture different aspects:

```
Head 1: Syntactic relationships (subject-verb)
Head 2: Semantic relationships (synonyms)
Head 3: Positional relationships (adjacent words)
Head 4: Long-range dependencies
...
```

### Architecture

```
                Input Embedding
                       ↓
        ┌──────────────┼──────────────┐
        ↓              ↓              ↓
    Head 1         Head 2    ...  Head h
    (Q,K,V)        (Q,K,V)        (Q,K,V)
        ↓              ↓              ↓
  Attention      Attention      Attention
        ↓              ↓              ↓
        └──────────────┴──────────────┘
                       ↓
                   Concatenate
                       ↓
                 Linear Layer
                       ↓
                Multi-Head Output
```

### Process

```python
# Typical configuration
d_model = 512      # Model dimension
h = 8              # Number of heads
d_k = d_v = 64     # Dimension per head (512/8)

For each head i:
1. Linear projections
   Q_i = X × W^Q_i  (shape: [seq_len, 64])
   K_i = X × W^K_i
   V_i = X × W^V_i

2. Scaled dot-product attention
   head_i = Attention(Q_i, K_i, V_i)

3. Concatenate all heads
   MultiHead = Concat(head_1, ..., head_8)
               (shape: [seq_len, 512])

4. Final linear projection
   Output = MultiHead × W^O
```

### Example: 4 Heads

```
Sentence: "The cat sat on the mat"

Head 1 learns: Noun relationships
  "cat" ← attends strongly to → "mat" (both nouns)

Head 2 learns: Verb-object relationships
  "sat" ← attends strongly to → "mat" (verb-object)

Head 3 learns: Article-noun relationships
  "The" ← attends strongly to → "cat" (article-noun)

Head 4 learns: Positional relationships
  Each word ← attends strongly to → adjacent words
```

### Benefits

1. **Diverse representations**: Different types of relationships
2. **Robustness**: Redundancy helps learning
3. **Parallel processing**: All heads computed simultaneously
4. **Richer context**: Multiple perspectives combined

## Feed-Forward Networks

### Purpose

After attention, each position is processed independently through a feed-forward network (FFN).

### Architecture

```
      Input from Attention
              ↓
      ┌─────────────┐
      │  Linear 1   │
      │  (expand)   │
      │ 512 → 2048  │
      └──────┬──────┘
             ↓
      ┌─────────────┐
      │   ReLU      │
      │ Activation  │
      └──────┬──────┘
             ↓
      ┌─────────────┐
      │  Linear 2   │
      │  (project)  │
      │ 2048 → 512  │
      └──────┬──────┘
             ↓
         Output
```

### Formula

```
FFN(x) = max(0, x·W₁ + b₁)·W₂ + b₂

Where:
W₁: [d_model × d_ff]    (e.g., 512 × 2048)
W₂: [d_ff × d_model]    (e.g., 2048 × 512)
d_ff: typically 4 × d_model
ReLU: max(0, x)
```

### Example

```python
Input vector: [0.5, -0.2, 0.8, 0.1] (d_model=4)

Step 1: Linear expansion (expand to d_ff=8)
W₁ × input = [-0.3, 0.7, 1.2, -0.5, 0.9, 0.4, -0.8, 0.2]

Step 2: ReLU activation
ReLU(x) = [0, 0.7, 1.2, 0, 0.9, 0.4, 0, 0.2]
         (negative values → 0)

Step 3: Linear projection (project back to d_model=4)
W₂ × activated = [0.6, -0.1, 0.9, 0.3]

Output: [0.6, -0.1, 0.9, 0.3]
```

### Why FFN?

1. **Non-linearity**: ReLU adds non-linear transformations
2. **Capacity**: Larger intermediate dimension increases model capacity
3. **Position-wise**: Applied independently to each position
4. **Feature transformation**: Learns complex feature combinations

## Layer Normalization and Residual Connections

### Residual Connections (Skip Connections)

Allow gradient flow and enable deep networks:

```
      Input (x)
         ↓
    ┌────┴────┐
    ↓         │ (skip)
Sublayer      │
(attention    │
or FFN)       │
    ↓         │
    └────┬────┘
         ↓
      Output = Sublayer(x) + x
```

**Example**:
```python
Input: [1.0, 0.5, -0.3]
Sublayer output: [0.2, 0.1, 0.4]
Final output: [1.2, 0.6, 0.1]  (sum of both)
```

### Layer Normalization

Normalizes activations across features:

```
LayerNorm(x) = γ × (x - μ) / σ + β

Where:
μ: mean of x
σ: standard deviation of x
γ, β: learned parameters
```

**Example**:
```python
Input: [1.0, 2.0, 3.0, 4.0]
Mean (μ): 2.5
Std (σ): 1.12

Normalized: [-1.34, -0.45, 0.45, 1.34]
            (mean=0, std=1)

After scale (γ) and shift (β):
Output: [0.2, 0.8, 1.4, 2.0]  (learned γ, β)
```

### Combined: Add & Norm

```
LayerNorm(x + Sublayer(x))

Example:
x: [1, 2, 3]
Sublayer(x): [0.5, 0.3, 0.2]
x + Sublayer(x): [1.5, 2.3, 3.2]
LayerNorm(...): [Normalized output]
```

**Benefits**:
- Stabilizes training
- Enables deeper networks (100+ layers)
- Faster convergence
- Better gradient flow

## Encoder vs Decoder vs Encoder-Decoder

### 1. Encoder-Only (e.g., BERT)

```
Input: "The cat sat on the mat"
   ↓
[Bidirectional Self-Attention]
   ↓ Can see entire input
Output: Contextual representations

Use Cases:
- Classification (sentiment, NER)
- Question answering
- Embeddings
```

**Example**:
```
Input: "Paris is the capital of France"
Task: Extract entities
Output: [Paris: LOCATION, France: LOCATION]
```

### 2. Decoder-Only (e.g., GPT, Llama)

```
Input: "The cat sat on the"
   ↓
[Masked Self-Attention]
   ↓ Can only see previous tokens
Output: "mat" (next token prediction)

Use Cases:
- Text generation
- Completion
- Chatbots
```

**Example**:
```
Input: "Once upon a time"
Output: "there was a brave knight who..."
```

### 3. Encoder-Decoder (e.g., T5, BART)

```
Encoder Input: "Hello, how are you?"
      ↓
[Bidirectional Attention]
      ↓
Encoder Output (context)
      ↓
Decoder Input: "<start>"
      ↓
[Masked Self-Attention + Cross-Attention to Encoder]
      ↓
Decoder Output: "Bonjour, comment allez-vous?"

Use Cases:
- Translation
- Summarization
- Question answering
```

### Comparison Table

| Feature | Encoder | Decoder | Encoder-Decoder |
|---------|---------|---------|-----------------|
| **Attention** | Bidirectional | Causal (masked) | Both |
| **Input visibility** | Full sequence | Previous tokens only | Full input, partial output |
| **Best for** | Understanding | Generation | Transformation |
| **Examples** | BERT, RoBERTa | GPT, Llama | T5, BART |
| **Tasks** | Classification | Completion | Translation |

## Key Takeaways

### Architecture Fundamentals

1. **Transformers use self-attention** instead of recurrence, enabling parallelization

2. **Positional encoding** adds position information since attention is position-agnostic

3. **QKV (Query-Key-Value)** mechanism is the core of self-attention

4. **Multi-head attention** captures different types of relationships simultaneously

5. **Feed-forward networks** add non-linearity and processing capacity

### Components

6. **Residual connections** enable deep networks (96+ layers in GPT-3)

7. **Layer normalization** stabilizes training and accelerates convergence

8. **Encoder-only** (BERT): Best for understanding and classification

9. **Decoder-only** (GPT): Best for text generation

10. **Encoder-decoder** (T5): Best for transformation tasks

### Practical Insights

11. **Attention complexity**: O(n²) in sequence length - challenging for long sequences

12. **Standard configuration**:
    - d_model = 512-12288
    - heads = 8-96
    - layers = 12-96
    - d_ff = 4 × d_model

13. **Scaling laws**: Larger models (more layers, wider dimensions) generally perform better
