## Introduction to Embeddings

### What are Embeddings?

**Embeddings** are dense vector representations of data (text, images, audio) in a continuous vector space, where semantically similar items are close together.

```
Text → Numbers that capture meaning

"cat" → [0.2, -0.5, 0.3, 0.1, ..., 0.4]  (768 dimensions)
"dog" → [0.3, -0.4, 0.2, 0.0, ..., 0.5]  (close to "cat")
"car" → [0.1, 0.6, -0.3, 0.8, ..., -0.2] (far from "cat")
```

### Why Embeddings?

**Traditional Representation (One-Hot Encoding)**:
```
Vocabulary: [cat, dog, car, house]

cat:   [1, 0, 0, 0]
dog:   [0, 1, 0, 0]
car:   [0, 0, 1, 0]
house: [0, 0, 0, 1]

Problems:
- Sparse (mostly zeros)
- No semantic meaning
- Vocabulary-dependent
- "cat" and "dog" appear equally different as "cat" and "car"
```

**Embedding Representation**:
```
cat:   [0.8, 0.2, -0.3, 0.5]
dog:   [0.7, 0.3, -0.2, 0.4]  ← Similar to cat
car:   [0.1, -0.5, 0.8, 0.2]  ← Different from cat
house: [-0.2, 0.7, 0.4, -0.1]

Benefits:
- Dense (all values meaningful)
- Captures semantic similarity
- Fixed size regardless of vocabulary
- "cat" and "dog" are closer than "cat" and "car"
```

### Key Properties

1. **Dimensionality**: Typically 128-4096 dimensions
2. **Density**: All values are meaningful (not sparse)
3. **Continuity**: Similar meaning → similar vectors
4. **Compositionality**: Can combine vectors meaningfully

## Word Embeddings

### Word2Vec (2013)

**Two Architectures**:

#### 1. CBOW (Continuous Bag of Words)
```
Predict center word from context

Context: "The cat sat on the"
        ↓
    [Average embeddings]
        ↓
    Predict: "mat"
```

#### 2. Skip-gram
```
Predict context from center word

Input: "cat"
        ↓
Predict context: ["the", "sat", "on", "mat"]
```

**Training Process**:
```python
# Skip-gram example
Input word: "cat"
Context window: 2

Sentence: "The quick brown cat jumped over"
                        ↑
Training pairs from "cat":
("cat", "brown")
("cat", "quick")
("cat", "jumped")
("cat", "over")

Model learns: cat_embedding should predict context_embeddings
```

**Result**: Word vectors capture semantic relationships
```
Vector arithmetic:
king - man + woman ≈ queen
Paris - France + Italy ≈ Rome
```

### GloVe (Global Vectors, 2014)

**Approach**: Matrix factorization on word co-occurrence statistics

```
Co-occurrence Matrix:
        cat  dog  mat  sat  car
cat:    0    50   80   60   5
dog:    50   0    20   15   3
mat:    80   20   0    90   2
...

GloVe learns embeddings that preserve these relationships
```

**Example**:
```python
# Pre-trained GloVe embeddings (50d)
cat:    [0.25, -0.45, 0.67, ..., -0.13]
dog:    [0.31, -0.39, 0.58, ..., -0.08]
kitten: [0.23, -0.43, 0.64, ..., -0.11]

Similarity:
cos_sim(cat, dog)    = 0.85  (high similarity)
cos_sim(cat, kitten) = 0.92  (very high similarity)
cos_sim(cat, car)    = 0.23  (low similarity)
```

### FastText (2016)

**Key Innovation**: Uses character n-grams

```
Word: "running"
N-grams: <ru, run, unn, nni, nin, ing, ng>

Benefits:
- Handles out-of-vocabulary words
- Captures morphology
- Works well for rare words

Example:
"running" = sum of embeddings for all n-grams
"run" + "running" share n-grams → similar embeddings
```

### Static vs Contextual Embeddings

**Static (Word2Vec, GloVe)**:
```
"bank" → [0.2, 0.5, -0.3, ...]  (same vector always)

Problem:
"river bank" and "bank account" get same embedding
```

**Contextual (BERT, GPT)**:
```
"I went to the river bank" → bank₁: [0.2, 0.3, 0.8, ...]
"I deposited money in the bank" → bank₂: [0.5, -0.2, 0.1, ...]

Different vectors based on context!
```

## Sentence and Document Embeddings

### Why Sentence Embeddings?

Word embeddings work for individual words, but we need representations for entire sentences/documents.

```
Sentence: "The cat sat on the mat"

Simple Average (naive):
avg([cat_emb, sat_emb, mat_emb, ...])

Problem: Loses word order and structure
"cat sat on mat" ≈ "mat sat on cat" (same average!)
```

### Sentence Embedding Methods

#### 1. Averaging Word Embeddings
```python
# Simple but loses information
sentence = "The cat sat"
embeddings = [emb("The"), emb("cat"), emb("sat")]
sentence_emb = mean(embeddings)
```

#### 2. Weighted Average (TF-IDF)
```python
# Weight important words more
sentence_emb = Σ (tfidf_weight(word) × emb(word))
```

#### 3. BERT [CLS] Token
```
[CLS] The cat sat on the mat [SEP]
  ↓
BERT processes with attention
  ↓
Extract [CLS] embedding → sentence representation
```

#### 4. Sentence Transformers (SBERT)
```
Fine-tuned BERT for sentence similarity

Architecture:
Sentence A → BERT → [CLS] → Embedding A
Sentence B → BERT → [CLS] → Embedding B
                              ↓
                    Compare similarity
```

### Example: Sentence Embeddings

```python
# Using Sentence-BERT
Sentence 1: "The dog is running in the park"
Embedding 1: [0.25, -0.45, 0.67, 0.12, ..., -0.13]  (384 dims)

Sentence 2: "A puppy plays outside"
Embedding 2: [0.28, -0.42, 0.63, 0.15, ..., -0.09]

Sentence 3: "Python is a programming language"
Embedding 3: [0.82, 0.15, -0.35, -0.52, ..., 0.47]

Cosine Similarity:
sim(Sent1, Sent2) = 0.87  (high - similar meaning)
sim(Sent1, Sent3) = 0.12  (low - different topics)
```

## Similarity Metrics

### 1. Cosine Similarity

**Most common for embeddings**

```
cos_sim(A, B) = (A · B) / (||A|| × ||B||)

Range: [-1, 1]
  1: Identical direction
  0: Orthogonal (unrelated)
 -1: Opposite direction
```

**Example**:
```python
A = [1, 2, 3]
B = [2, 4, 6]  (scaled version of A)

A · B = 1×2 + 2×4 + 3×6 = 2 + 8 + 18 = 28
||A|| = √(1² + 2² + 3²) = √14 ≈ 3.74
||B|| = √(2² + 4² + 6²) = √56 ≈ 7.48

cos_sim = 28 / (3.74 × 7.48) = 28 / 28 = 1.0
(Perfect similarity - same direction!)
```

### 2. Euclidean Distance

```
distance(A, B) = √(Σ (Aᵢ - Bᵢ)²)

Range: [0, ∞]
  0: Identical
  ∞: Very different
```

**Example**:
```python
A = [1, 2, 3]
B = [4, 5, 6]

distance = √[(1-4)² + (2-5)² + (3-6)²]
         = √[9 + 9 + 9]
         = √27 ≈ 5.20
```

### 3. Dot Product

```
dot(A, B) = Σ (Aᵢ × Bᵢ)

Range: (-∞, ∞)
Higher = more similar (if normalized)
```

**Example**:
```python
A = [1, 2, 3]
B = [2, 3, 4]

dot = 1×2 + 2×3 + 3×4 = 2 + 6 + 12 = 20
```

### Comparison

```
Sentence pairs with embeddings:

Pair 1: "dog" vs "puppy"
  Cosine: 0.89
  Euclidean: 2.3
  Dot product: 245

Pair 2: "dog" vs "cat"
  Cosine: 0.75
  Euclidean: 3.8
  Dot product: 198

Pair 3: "dog" vs "computer"
  Cosine: 0.12
  Euclidean: 12.5
  Dot product: 45

Cosine similarity is most commonly used for semantic similarity
```

## Vector Databases

### What is a Vector Database?

A **vector database** is a specialized database optimized for storing and querying high-dimensional vectors (embeddings).

### Why Vector Databases?

**Traditional Database**:
```sql
SELECT * FROM documents
WHERE title = 'Machine Learning'

Problem: Exact match only, no semantic search
```

**Vector Database**:
```python
query_embedding = embed("Machine Learning")
results = vector_db.search(
    query_embedding,
    top_k=5
)

Returns:
1. "Introduction to ML" (similarity: 0.95)
2. "Deep Learning Basics" (similarity: 0.87)
3. "AI Fundamentals" (similarity: 0.82)
...

Benefit: Semantic similarity search!
```

### Key Features

1. **Similarity Search**: Find nearest neighbors efficiently
2. **Filtering**: Combine vector search with metadata filters
3. **Scalability**: Handle millions to billions of vectors
4. **Performance**: Millisecond query times

### Vector Search Algorithms

#### 1. Exact Search (Brute Force)
```
Compare query to every vector in database

Process:
For each vector in database:
    compute similarity(query, vector)
Return top-k most similar

Complexity: O(n × d)
Pro: Perfect accuracy
Con: Slow for large datasets
```

#### 2. Approximate Nearest Neighbor (ANN)

**HNSW (Hierarchical Navigable Small World)**:
```
Multi-layer graph structure

Layer 2:  ●--------●--------●
          |        |        |
Layer 1:  ●--●--●--●--●--●--●
          |  |  |  |  |  |  |
Layer 0:  ●-●-●-●-●-●-●-●-●-●-●

Search: Start at top layer, navigate down
Complexity: O(log n)
Pro: Fast
Con: Approximate (99%+ accuracy typically)
```

**IVF (Inverted File Index)**:
```
1. Cluster vectors into groups
2. Search only relevant clusters

Clusters:
  [Dogs, Cats, Pets] → Cluster 1
  [Cars, Bikes] → Cluster 2
  [Code, Python] → Cluster 3

Query: "puppy" → Search only Cluster 1

Pro: Very fast
Con: May miss results in other clusters
```

### Vector Database Operations

#### 1. Insert
```python
# Add vectors to database
vector_db.insert(
    vectors=[[0.1, 0.2, ...], [0.3, 0.4, ...]],
    metadata=[{"text": "doc1"}, {"text": "doc2"}],
    ids=["id1", "id2"]
)
```

#### 2. Search
```python
# Find similar vectors
results = vector_db.search(
    query_vector=[0.15, 0.25, ...],
    top_k=10,
    filter={"category": "technology"}
)
```

#### 3. Update
```python
# Update existing vector
vector_db.update(
    id="id1",
    vector=[0.2, 0.3, ...]
)
```

#### 4. Delete
```python
# Remove vector
vector_db.delete(id="id1")
```

## Popular Vector Database Solutions

### 1. Pinecone

**Type**: Managed cloud service

```python
import pinecone

# Initialize
pinecone.init(api_key="key")
index = pinecone.Index("my-index")

# Insert
index.upsert(vectors=[
    ("id1", [0.1, 0.2, ...], {"text": "doc1"}),
    ("id2", [0.3, 0.4, ...], {"text": "doc2"})
])

# Query
results = index.query(
    vector=[0.15, 0.25, ...],
    top_k=5
)
```

**Features**:
- Fully managed
- Auto-scaling
- Real-time updates
- Filtering support

### 2. Weaviate

**Type**: Open-source, self-hosted or cloud

```python
import weaviate

client = weaviate.Client("http://localhost:8080")

# Create schema
client.schema.create_class({
    "class": "Document",
    "vectorizer": "text2vec-transformers"
})

# Insert
client.data_object.create(
    data_object={"text": "Machine learning doc"},
    class_name="Document"
)

# Query
results = client.query.get("Document", ["text"])\
    .with_near_text({"concepts": ["AI"]})\
    .with_limit(5)\
    .do()
```

**Features**:
- GraphQL API
- Built-in vectorization
- Hybrid search
- Multi-tenancy

### 3. Milvus

**Type**: Open-source, highly scalable

```python
from pymilvus import connections, Collection

# Connect
connections.connect("default", host="localhost", port="19530")

# Create collection
collection = Collection("documents")

# Insert
collection.insert([
    [0.1, 0.2, ...],
    [0.3, 0.4, ...]
])

# Search
results = collection.search(
    data=[[0.15, 0.25, ...]],
    anns_field="embedding",
    param={"metric_type": "L2", "params": {"nprobe": 10}},
    limit=5
)
```

**Features**:
- Extremely scalable
- GPU support
- Multiple index types
- Kubernetes-native

### 4. Chroma

**Type**: Open-source, embedded or server

```python
import chromadb

client = chromadb.Client()
collection = client.create_collection("my_collection")

# Add documents (auto-embedding)
collection.add(
    documents=["doc1 text", "doc2 text"],
    metadatas=[{"source": "web"}, {"source": "pdf"}],
    ids=["id1", "id2"]
)

# Query
results = collection.query(
    query_texts=["machine learning"],
    n_results=5
)
```

**Features**:
- Easy to use
- Auto-embedding
- In-memory or persistent
- Python-first

### 5. Qdrant

**Type**: Open-source, Rust-based

```python
from qdrant_client import QdrantClient

client = QdrantClient("localhost", port=6333)

# Create collection
client.create_collection(
    collection_name="documents",
    vectors_config={"size": 384, "distance": "Cosine"}
)

# Insert
client.upsert(
    collection_name="documents",
    points=[
        {"id": 1, "vector": [0.1, 0.2, ...], "payload": {"text": "doc1"}}
    ]
)

# Search
results = client.search(
    collection_name="documents",
    query_vector=[0.15, 0.25, ...],
    limit=5
)
```

**Features**:
- High performance (Rust)
- Payload filtering
- Quantization support
- Extended filtering

## Applications and Use Cases

### 1. Semantic Search

```python
# Index documents
docs = [
    "Introduction to machine learning",
    "Deep learning with neural networks",
    "Python programming basics",
    "Advanced Python techniques"
]

embeddings = [embed(doc) for doc in docs]
vector_db.insert(embeddings, docs)

# Search
query = "AI and neural nets"
query_emb = embed(query)
results = vector_db.search(query_emb, top_k=2)

Output:
1. "Deep learning with neural networks" (0.89)
2. "Introduction to machine learning" (0.76)
```

### 2. Recommendation Systems

```python
# User likes document
user_likes = "Introduction to Python"
user_emb = embed(user_likes)

# Find similar documents
recommendations = vector_db.search(user_emb, top_k=5)

Output:
1. "Python programming basics" (0.92)
2. "Advanced Python techniques" (0.84)
3. "Python for data science" (0.79)
```

### 3. Question Answering

```python
# Knowledge base
kb = [
    "Paris is the capital of France",
    "Python was created by Guido van Rossum",
    "NVIDIA manufactures GPUs"
]

kb_embs = [embed(fact) for fact in kb]
vector_db.insert(kb_embs, kb)

# Query
question = "Who created Python?"
q_emb = embed(question)
relevant_facts = vector_db.search(q_emb, top_k=1)

Output: "Python was created by Guido van Rossum"
```

### 4. Duplicate Detection

```python
# New document
new_doc = "Machine learning introduction"
new_emb = embed(new_doc)

# Check for duplicates
similar = vector_db.search(new_emb, top_k=1)

if similar[0].score > 0.95:
    print("Duplicate detected!")
```

### 5. RAG (Retrieval-Augmented Generation)

```python
# User query
query = "What is transformer architecture?"

# Retrieve relevant documents
query_emb = embed(query)
relevant_docs = vector_db.search(query_emb, top_k=3)

# Generate answer with LLM
context = "\n".join([doc.text for doc in relevant_docs])
prompt = f"Context: {context}\n\nQuestion: {query}\nAnswer:"
answer = llm.generate(prompt)
```

## Key Takeaways

### Embeddings Fundamentals

1. **Embeddings convert text to dense vectors** capturing semantic meaning

2. **Similar text has similar embeddings** (measured by cosine similarity)

3. **Word2Vec and GloVe** are static embeddings (one vector per word)

4. **BERT and transformers** produce contextual embeddings (different vectors based on context)

5. **Sentence embeddings** represent entire sentences/documents as single vectors

### Vector Databases

6. **Vector databases enable semantic search** beyond keyword matching

7. **ANN algorithms** (HNSW, IVF) provide fast approximate search

8. **Cosine similarity** is the most common metric for text embeddings

9. **Popular solutions**: Pinecone (managed), Weaviate (hybrid), Milvus (scalable), Chroma (easy)

10. **Applications**: Search, recommendations, QA, RAG, duplicate detection

### Practical Considerations

11. **Higher dimensions** (768-1536) generally better quality but slower

12. **Trade-off**: Accuracy vs speed (exact vs approximate search)

13. **Metadata filtering** combines semantic search with structured data

14. **Normalization** important for cosine similarity
