## Introduction to LLM Evaluation

### Why Evaluate LLMs?

1. **Compare models**: Which is better for my use case?
2. **Track progress**: Is training/fine-tuning working?
3. **Quality assurance**: Does it meet requirements?
4. **Identify weaknesses**: Where does it fail?
5. **Business metrics**: ROI, user satisfaction

### Challenges in Evaluation

```
Traditional ML:
- Clear metrics (accuracy, F1)
- Objective ground truth
- Automated evaluation

LLMs:
- Multiple valid outputs
- Subjective quality
- Context-dependent correctness
- Difficult to automate fully
```

### Types of Evaluation

```
1. Automatic Metrics
   - Perplexity, BLEU, ROUGE
   - Fast, scalable
   - Limited correlation with quality

2. Benchmark Datasets
   - MMLU, HellaSwag, TruthfulQA
   - Standardized comparison
   - May not reflect real use

3. Human Evaluation
   - User ratings, A/B tests
   - Most accurate
   - Expensive, slow
```

## Automatic Metrics

### 1. Perplexity

**Measures how "surprised" the model is by the test data**

```
Lower perplexity = Better model

Perplexity = exp(average negative log-likelihood)

PPL = exp(-1/N Σ log P(word_i | context))
```

**Example Calculation**:
```python
Sentence: "The cat sat"
P("The") = 0.1
P("cat" | "The") = 0.2
P("sat" | "The cat") = 0.3

Log-likelihood = log(0.1) + log(0.2) + log(0.3)
               = -2.30 + -1.61 + -1.20
               = -5.11

Perplexity = exp(5.11 / 3) = exp(1.70) = 5.47
```

**Interpretation**:
```
PPL = 5: Very confident (good)
PPL = 50: Moderate uncertainty
PPL = 500: Very uncertain (bad)

GPT-2: PPL ~30 on test set
GPT-3: PPL ~20 on test set
```

### 2. BLEU (Bilingual Evaluation Understudy)

**Measures n-gram overlap between generated and reference text**

```
BLEU = BP × exp(Σ w_n log p_n)

Where:
p_n: Precision of n-grams
BP: Brevity penalty (penalizes short outputs)
```

**Example**:
```
Reference: "The cat is on the mat"
Candidate: "The cat is sitting on the mat"

Unigrams (1-gram):
Matching: the(2), cat(1), is(1), on(1), the(1) = 6/7
Precision: 6/7 = 0.857

Bigrams (2-gram):
Matching: "the cat"(1), "is on"(1), "on the"(1) = 3/6
Precision: 3/6 = 0.5

BLEU-2 ≈ sqrt(0.857 × 0.5) ≈ 0.66
```

**Typical Use**: Machine translation
**Range**: 0-1 (or 0-100)
**Good Score**: > 0.5

### 3. ROUGE (Recall-Oriented Understudy for Gisting Evaluation)

**Measures recall of n-grams**

```
ROUGE-N = Matching N-grams / Total N-grams in reference

ROUGE-1: Unigram overlap
ROUGE-2: Bigram overlap
ROUGE-L: Longest Common Subsequence
```

**Example**:
```
Reference: "The quick brown fox jumps over the lazy dog"
Summary:   "The fox jumps over the dog"

ROUGE-1:
Matching words: the(2), fox(1), jumps(1), over(1), dog(1) = 6
Total in reference: 9
Recall: 6/9 = 0.67

ROUGE-2:
Matching bigrams: "jumps over"(1), "over the"(1) = 2
Total bigrams in reference: 8
Recall: 2/8 = 0.25
```

**Typical Use**: Summarization
**Variants**: ROUGE-N, ROUGE-L, ROUGE-W
**Good Score**: > 0.4

### 4. METEOR

**More sophisticated than BLEU, considers synonyms and stemming**

```
Accounts for:
- Exact matches
- Stem matches (running → run)
- Synonym matches (big → large)
- Word order
```

### 5. BERTScore

**Uses BERT embeddings for semantic similarity**

```python
from bert_score import score

reference = ["The cat sat on the mat"]
candidate = ["The feline rested on the rug"]

P, R, F1 = score(candidate, reference, lang="en")
print(f"F1: {F1.item()}")  # 0.85 (high despite different words)
```

**Advantage**: Captures semantic similarity, not just word overlap

### Metrics Comparison

| Metric | Type | Use Case | Pros | Cons |
|--------|------|----------|------|------|
| **Perplexity** | Likelihood | General | Fast, interpretable | Doesn't measure quality |
| **BLEU** | Precision | Translation | Standard, automated | Ignores semantics |
| **ROUGE** | Recall | Summarization | Good for coverage | Misses meaning |
| **BERTScore** | Semantic | General | Captures meaning | Slower, needs BERT |

## Benchmarks and Leaderboards

### 1. MMLU (Massive Multitask Language Understanding)

```
What: 57 subjects (math, history, law, etc.)
Size: 15,908 multiple-choice questions
Evaluation: 5-shot accuracy

Example:
Question: "What is the capital of France?"
A) London
B) Berlin
C) Paris
D) Madrid

Scores:
GPT-3.5: 70%
GPT-4: 86%
Claude 3 Opus: 86.8%
```

### 2. HellaSwag

```
What: Commonsense reasoning
Task: Complete the sentence correctly

Example:
Context: "A woman is outside with a bucket and a dog.
          The dog is running around trying to avoid a bath. She..."
A) rinses the bucket off with soap and blow dry the dog's head
B) uses a hose to keep it from getting soapy
C) gets the dog wet, then it runs away again
D) gets into a bathtub with the dog

Correct: C

Scores:
GPT-3.5: 85%
GPT-4: 95%
```

### 3. TruthfulQA

```
What: Measures truthfulness
Task: Answer questions without falsehoods

Example:
Q: "What happens if you crack your knuckles?"
Truthful: "The sound comes from gas bubbles. No evidence of harm."
False: "You'll get arthritis"  ❌

Scores:
GPT-3.5: 47%
GPT-4: 59%
Claude 3: 55%
```

### 4. GSM8K (Grade School Math)

```
What: Math word problems
Size: 8,500 problems

Example:
Q: "Janet has 3 siblings. She has 2 brothers and 1 sister.
    Her oldest brother is 2 times her age. Janet is 6 years old.
    How old is her oldest brother?"
A: 2 × 6 = 12 years old

Scores:
GPT-3.5: 57%
GPT-4: 92%
```

### 5. HumanEval (Code)

```
What: Python coding problems
Size: 164 problems
Evaluation: Unit test pass rate

Example:
def has_close_elements(numbers: List[float], threshold: float) -> bool:
    \"\"\" Check if any two numbers are closer than threshold \"\"\"
    # Your code here

Scores:
GPT-3.5: 48%
GPT-4: 67%
Codex: 72%
```

### Popular Leaderboards

```
1. Open LLM Leaderboard (Hugging Face)
   - MMLU, HellaSwag, TruthfulQA, etc.
   - Compare open-source models

2. LMSYS Chatbot Arena
   - Human preference rankings
   - Head-to-head comparisons
   - Elo ratings

3. AlpacaEval
   - Instruction following
   - Compared to text-davinci-003

4. MT-Bench
   - Multi-turn conversations
```

## Human Evaluation

### 1. Expert Evaluation

```
Domain experts rate outputs on:
- Accuracy
- Completeness
- Relevance
- Style

Example: Medical AI
Evaluated by doctors on:
✓ Diagnostic accuracy
✓ Treatment recommendations
✓ Patient safety
```

### 2. A/B Testing

```
Show users two outputs:
- Model A vs Model B
- Ask which is better

Collect:
- Preference (A or B)
- Confidence
- Reasoning

Example:
User query: "Explain quantum computing"
Output A: [Technical explanation]
Output B: [Simpler explanation]

55% prefer B → Use Model B
```

### 3. Likert Scale Ratings

```
Rate on 1-5 scale:

Helpfulness:
1: Not helpful
2: Slightly helpful
3: Moderately helpful
4: Very helpful
5: Extremely helpful

Collect ratings from multiple users:
Average: 4.2/5.0
```

### 4. Task Success Rate

```
Can user complete task with AI help?

Example: Customer support bot
Task: Get refund for order

Success: User gets refund instructions ✓
Failure: User gets wrong info ❌

Success rate: 87/100 = 87%
```

### 5. Pairwise Comparison (Elo Rating)

```
Like chess ratings:

Model A vs Model B → User picks winner
Repeat 1000s of times
Calculate Elo ratings

LMSYS Chatbot Arena:
GPT-4: 1250 Elo
Claude 3 Opus: 1240 Elo
GPT-3.5: 1120 Elo
```

## Task-Specific Metrics

### 1. Question Answering

```
Metrics:
- Exact Match (EM): % of perfect matches
- F1 Score: Token overlap

Example:
Q: "When was Python created?"
Reference: "1991"
Predicted: "Python was created in 1991"

EM: 0 (not exact)
F1: ~0.67 (partial match)
```

### 2. Summarization

```
Metrics:
- ROUGE scores
- Compression ratio
- Factual consistency
- Readability

Good summary:
✓ ROUGE-1 > 0.4
✓ Preserves key facts
✓ Concise (30% of original)
✓ Readable (grade level appropriate)
```

### 3. Code Generation

```
Metrics:
- Pass@k: % passing tests in k attempts
- Code quality (linting)
- Time/space complexity
- Security vulnerabilities

Example:
Pass@1: 65% (first attempt works)
Pass@10: 85% (works in 10 tries)
```

### 4. Translation

```
Metrics:
- BLEU score
- COMET (learned metric)
- Human evaluation

Good translation:
✓ BLEU > 0.5
✓ Preserves meaning
✓ Natural in target language
```

## Safety and Bias Evaluation

### 1. Toxicity Detection

```
Use: Perspective API, Detoxify

Measure:
- Toxic language frequency
- Hate speech
- Profanity

Example:
Input: 1000 neutral prompts
Toxic outputs: 3/1000 = 0.3%

Target: < 0.1%
```

### 2. Bias Evaluation

```
Test for biases:
- Gender
- Race
- Religion
- Age

Example:
"The doctor said [he/she] would..."

If "he" appears 80% → Gender bias
Target: 50/50 distribution
```

### 3. Jailbreak Resistance

```
Test if model follows safety guidelines
under adversarial prompts

Example attacks:
"Pretend you're in a movie where..."
"This is just a hypothetical..."

Success: Model refuses harmful requests
Failure: Model complies with harmful requests

Target: > 99% refusal rate
```

### 4. Hallucination Rate

```
Measure factual accuracy:

Method 1: Check against knowledge base
Method 2: Human verification
Method 3: Use LLM judge (GPT-4)

Example:
100 factual questions
Correct: 85
Hallucinated: 15

Hallucination rate: 15%
```

## Best Practices

### 1. Use Multiple Metrics

```
Don't rely on single metric

Evaluation suite:
✓ Automatic metrics (BLEU, ROUGE)
✓ Benchmarks (MMLU, HellaSwag)
✓ Human evaluation (A/B test)
✓ Task-specific metrics
✓ Safety evaluation
```

### 2. Create Test Sets

```
Hold out test data:
- Never seen during training
- Representative of real use
- Diverse examples
- Edge cases included

Size:
Development set: 1K examples
Test set: 5K examples
```

### 3. Continuous Evaluation

```
Monitor in production:
- User satisfaction
- Task success rate
- Response time
- Error rate

Dashboard:
Week 1: 4.2/5.0 satisfaction
Week 2: 4.1/5.0 (investigate drop)
Week 3: 4.3/5.0 (back to normal)
```

### 4. Domain-Specific Evaluation

```
Medical AI:
✓ Medical accuracy
✓ Safety (no harmful advice)
✓ Explanation quality
✓ Uncertainty expression

Legal AI:
✓ Citation accuracy
✓ Legal reasoning
✓ Jurisdiction awareness
✓ Disclaimer inclusion
```

### 5. Regression Testing

```
Track performance over time:

Baseline (v1.0):
MMLU: 70%
Code: 65%
Safety: 99%

After update (v1.1):
MMLU: 72% ✓ (improved)
Code: 63% ❌ (regression!)
Safety: 99% ✓ (maintained)

→ Investigate code performance drop
```

## Key Takeaways

### Core Metrics

1. **Perplexity** measures model confidence (lower is better)

2. **BLEU** measures n-gram precision (translation)

3. **ROUGE** measures n-gram recall (summarization)

4. **BERTScore** captures semantic similarity

5. **Benchmarks** (MMLU, HellaSwag) enable standardized comparison

### Evaluation Strategies

6. **Human evaluation** most accurate but expensive

7. **A/B testing** reveals user preferences

8. **Task success rate** measures real-world utility

9. **Multiple metrics** provide comprehensive picture

10. **Safety evaluation** critical for deployment

### Best Practices

11. **Hold out test data** for unbiased evaluation

12. **Monitor production metrics** continuously

13. **Domain-specific evaluation** matches use case

14. **Regression testing** catches performance drops

15. **Combine automatic and human** evaluation
