## Introduction to Prompt Engineering

### What is Prompt Engineering?

**Prompt Engineering** is the practice of designing and optimizing inputs (prompts) to language models to elicit desired outputs.

```
Prompt = Instruction + Context + Input + Output Format

Example:
"Translate the following English text to French:
Text: Hello, how are you?
Translation:"
```

### Why Prompt Engineering Matters

```
Poor Prompt:
"Write about AI"

→ Vague output, unpredictable

Good Prompt:
"Write a 3-paragraph explanation of how transformers work in AI,
targeted at software engineers with no ML background.
Include: 1) What problem they solve, 2) Key components, 3) Why they're important."

→ Specific, structured, useful output
```

### Key Benefits

1. **No training required**: Immediate results without model updates
2. **Cost-effective**: Cheaper than fine-tuning
3. **Flexible**: Easy to iterate and experiment
4. **Task adaptation**: Same model, different tasks via prompts

## Basic Prompting Principles

### 1. Be Specific and Clear

```
❌ Bad: "Tell me about Python"

✅ Good: "Explain Python list comprehensions with 3 examples,
including filtering and transformation operations."
```

### 2. Provide Context

```
❌ Bad: "Is this good?"

✅ Good: "As a senior software engineer reviewing this code:
[code block]
Is this implementation efficient and following best practices?"
```

### 3. Specify Format

```
❌ Bad: "List programming languages"

✅ Good: "List 5 programming languages in JSON format:
{
  \"languages\": [
    {\"name\": \"Python\", \"use_case\": \"...\"}
  ]
}"
```

### 4. Give Examples

```
❌ Bad: "Classify sentiment"

✅ Good: "Classify the sentiment as positive, negative, or neutral.

Examples:
'I love this!' → Positive
'Terrible experience' → Negative
'It was okay' → Neutral

Text: 'The product exceeded my expectations'
Sentiment:"
```

### 5. Break Down Complex Tasks

```
❌ Bad: "Analyze this document and give me insights"

✅ Good:
"Analyze the following document in 3 steps:
1. Identify the main topics (list 3-5)
2. Summarize key points for each topic
3. Provide actionable recommendations

Document: [text]"
```

## Zero-Shot Prompting

### Definition

Asking the model to perform a task without any examples.

### Examples

#### Task: Classification
```
Prompt:
"Classify the following text as spam or not spam:
'Congratulations! You've won $1000! Click here now!'
Classification:"

Output: "Spam"
```

#### Task: Extraction
```
Prompt:
"Extract the person's name, email, and phone number from:
'Contact John Smith at john@example.com or call 555-0123'

Output in JSON format."

Output:
{
  "name": "John Smith",
  "email": "john@example.com",
  "phone": "555-0123"
}
```

#### Task: Summarization
```
Prompt:
"Summarize the following article in 2 sentences:
[article text]

Summary:"

Output: [2-sentence summary]
```

### When to Use Zero-Shot

- Simple, well-defined tasks
- Common operations (translation, summarization)
- When examples are hard to provide
- Quick prototyping

## Few-Shot Prompting

### Definition

Providing examples to demonstrate the desired behavior.

### Standard Few-Shot

```
Prompt:
"Convert the product description to a catchy tagline.

Product: Running shoes
Tagline: Run Further, Faster

Product: Noise-cancelling headphones
Tagline: Silence the World, Amplify Your Music

Product: Smartwatch
Tagline: Time Redefined, Life Optimized

Product: Wireless keyboard
Tagline:"

Output: "Type Freely, Work Seamlessly"
```

### Few-Shot for Complex Tasks

#### Example: Code Generation
```
Prompt:
"Convert natural language to SQL.

Q: Get all users
SQL: SELECT * FROM users;

Q: Get users older than 25
SQL: SELECT * FROM users WHERE age > 25;

Q: Get top 10 highest paid employees
SQL: SELECT * FROM employees ORDER BY salary DESC LIMIT 10;

Q: Get average salary by department
SQL:"

Output: "SELECT department, AVG(salary) FROM employees GROUP BY department;"
```

#### Example: Structured Extraction
```
Prompt:
"Extract structured data from text.

Text: 'Apple announced iPhone 15 on September 12, 2023 for $799'
Output: {\"company\": \"Apple\", \"product\": \"iPhone 15\",
         \"date\": \"2023-09-12\", \"price\": 799}

Text: 'Tesla revealed Cybertruck on November 30, 2023 starting at $60,990'
Output: {\"company\": \"Tesla\", \"product\": \"Cybertruck\",
         \"date\": \"2023-11-30\", \"price\": 60990}

Text: 'Google launched Pixel 8 on October 4, 2023 priced at $699'
Output:"

Output: {\"company\": \"Google\", \"product\": \"Pixel 8\",
         \"date\": \"2023-10-04\", \"price\": 699}
```

### How Many Examples?

```
1-shot: Minimal guidance, may work for simple tasks
3-shot: Standard, balances context and performance
5-shot: Better for complex tasks
10+ shot: Diminishing returns, uses more context
```

## Chain-of-Thought Prompting

### Basic Chain-of-Thought (CoT)

Encourage step-by-step reasoning by showing the thinking process.

```
Prompt:
"Q: Roger has 5 tennis balls. He buys 2 more cans of tennis balls.
Each can has 3 tennis balls. How many tennis balls does he have now?

A: Let's think step by step.
Roger started with 5 balls.
2 cans of 3 balls each is 2 × 3 = 6 balls.
5 + 6 = 11 balls.
The answer is 11.

Q: The cafeteria had 23 apples. If they used 20 to make lunch and
bought 6 more, how many do they have?

A:"

Output:
"Let's think step by step.
The cafeteria started with 23 apples.
They used 20, so 23 - 20 = 3 apples left.
They bought 6 more, so 3 + 6 = 9 apples.
The answer is 9."
```

### Zero-Shot Chain-of-Thought

Simply add "Let's think step by step" to the prompt.

```
Prompt:
"Q: If a train travels 120 miles in 2 hours, how long will it
take to travel 300 miles at the same speed?

A: Let's think step by step."

Output:
"First, find the speed: 120 miles / 2 hours = 60 mph.
Then, calculate time: 300 miles / 60 mph = 5 hours.
The answer is 5 hours."
```

### Chain-of-Thought for Complex Reasoning

```
Prompt:
"Analyze whether this startup idea is viable. Think through:
1. Market demand
2. Competition
3. Technical feasibility
4. Monetization potential

Idea: AI-powered meal planning app for people with diabetes.

Analysis: Let's think step by step."

Output:
"1. Market demand: Large addressable market - ~37M diabetics in US,
   need for specialized meal planning.
2. Competition: Existing apps like MySugr, but none specifically
   focused on meal planning with AI personalization.
3. Technical feasibility: NLP for recipe parsing, ML for personalization,
   integration with glucose monitors - all technically achievable.
4. Monetization: Subscription model ($10-15/month), potential B2B
   with insurance companies.
Overall: Viable, with strong demand and differentiation."
```

### Self-Consistency

Generate multiple reasoning paths and pick most consistent answer.

```
Prompt (asked 5 times):
"Q: A store had 25 apples. They sold some and have 17 left.
How many did they sell? Let's think step by step."

Reasoning Path 1: 25 - 17 = 8 apples sold ✓
Reasoning Path 2: 25 - 17 = 8 apples sold ✓
Reasoning Path 3: 25 - 17 = 8 apples sold ✓
Reasoning Path 4: 17 - 25 = ... (error)
Reasoning Path 5: 25 - 17 = 8 apples sold ✓

Most consistent answer: 8 (appears 4/5 times)
```

## Advanced Techniques

### 1. ReAct (Reasoning + Acting)

Interleave reasoning and actions.

```
Prompt:
"Answer the question by thinking and using tools.

Question: What is the population of the capital of France?

Thought 1: I need to find the capital of France.
Action 1: Search[capital of France]
Observation 1: Paris is the capital of France.

Thought 2: Now I need the population of Paris.
Action 2: Search[population of Paris]
Observation 2: Paris has approximately 2.2 million inhabitants.

Thought 3: I have the answer.
Answer: The population of Paris, the capital of France, is approximately
2.2 million."
```

### 2. Tree of Thoughts

Explore multiple reasoning paths.

```
Prompt:
"Solve this problem by exploring different approaches:

Problem: Design a sustainable urban transportation system.

Approach 1: Public transit focused
- Pros: Lower emissions, efficient
- Cons: Initial cost, route coverage

Approach 2: Electric vehicle infrastructure
- Pros: Flexible, leverages existing roads
- Cons: Battery disposal, charging time

Approach 3: Bike/pedestrian infrastructure
- Pros: Healthy, zero emissions
- Cons: Weather dependent, limited distance

Best approach: Integrated system combining all three..."
```

### 3. Generated Knowledge

Generate relevant knowledge before answering.

```
Prompt:
"First, generate relevant knowledge about transformers in ML.
Then answer the question using that knowledge.

Knowledge:
- Transformers use self-attention mechanisms
- Introduced in 'Attention is All You Need' (2017)
- Key components: encoder, decoder, attention, feed-forward
- Enable parallel processing unlike RNNs

Question: Why are transformers better than RNNs for NLP?

Answer: Based on the knowledge above, transformers are better than
RNNs because they enable parallel processing (RNNs are sequential),
use self-attention to capture long-range dependencies more effectively,
and scale better to large datasets."
```

### 4. Automatic Prompt Optimization

Use LLM to improve prompts.

```
Prompt to LLM:
"I'm trying to get an LLM to summarize articles well, but my current
prompt often produces summaries that are too short. Here's my prompt:

'Summarize this article: [text]'

Suggest an improved prompt."

Output:
"Summarize the following article in 3-5 bullet points, capturing:
1. Main topic and key findings
2. Important supporting details
3. Implications or conclusions

Aim for approximately 100-150 words total.

Article: [text]

Summary:"
```

### 5. Constrained Generation

Force specific output formats.

```
Prompt:
"Generate a product review following this EXACT structure:

Rating: [1-5 stars]
Pros: [3 bullet points]
Cons: [2 bullet points]
Recommendation: [Yes/No with reason]

Product: Wireless earbuds XYZ"

Output:
Rating: 4 stars
Pros:
- Excellent sound quality
- Long battery life (8 hours)
- Comfortable fit
Cons:
- Expensive ($200)
- Limited color options
Recommendation: Yes - Great for audiophiles willing to pay premium
```

## Prompt Patterns and Templates

### 1. Instruction Template

```
Role: [Define the AI's role]
Task: [What to do]
Context: [Background information]
Constraints: [Limitations/requirements]
Output format: [How to structure output]

Example:
Role: You are a senior Python developer
Task: Review this code for bugs and improvements
Context: This is a web scraping script for e-commerce
Constraints: Follow PEP 8, suggest type hints
Output format: List issues with severity (high/medium/low)

[Code block]
```

### 2. QA Template

```
Context: [Provide background]
Question: [The actual question]
Format: [How to answer]

Example:
Context: You are helping a data scientist choose tools.
Question: Should I use PostgreSQL or MongoDB for this project?
Project needs: Frequent schema changes, 10M documents, complex queries
Format: Provide recommendation with 3 pros and 3 cons for each
```

### 3. Persona Template

```
"You are [persona] with [traits].
[User input]
Respond as [persona] would."

Example:
"You are Sherlock Holmes, brilliant detective known for logical deduction
and attention to detail.
Analyze this crime scene description and deduce what happened.
Respond as Sherlock would, with step-by-step reasoning."
```

### 4. Completion Template

```
"Complete the following [type of content]:

[Beginning]

Continue in the same style."

Example:
"Complete the following Python function:

def fibonacci(n):
    '''Calculate nth Fibonacci number'''
    # Your code here
"
```

## Best Practices

### 1. Iterative Refinement

```
Version 1: "Summarize this"
→ Too vague

Version 2: "Summarize this article in 100 words"
→ Better, but generic

Version 3: "Summarize this technical article in 100 words,
focusing on methodology and key findings, for software engineers"
→ Good, specific

Version 4: "Summarize this technical article using:
- 1 sentence overview
- 3 bullet points for methodology
- 2 bullet points for key findings
Target audience: Software engineers
Total: ~100 words"
→ Excellent, structured
```

### 2. Test with Edge Cases

```
Test Case 1: Empty input
Test Case 2: Very long input
Test Case 3: Ambiguous input
Test Case 4: Multiple valid answers
Test Case 5: Adversarial input

Example:
Task: Extract email addresses

Test with:
- "Contact us at" (no email)
- Text with 50 email addresses
- "email-like-string@notanemail" (invalid)
- "john@example.com and jane@example.com" (multiple)
```

### 3. Use Delimiters

```
Use delimiters to separate sections:

Prompt:
'''
Analyze the following code:

```python
def process_data(data):
    return [x * 2 for x in data]
```

Provide:
1. Time complexity
2. Space complexity
3. Potential improvements
'''
```

### 4. Temperature and Sampling

```
Temperature 0.0:
- Deterministic, consistent
- Use for: Factual QA, code generation, extraction

Temperature 0.7:
- Balanced creativity and consistency
- Use for: Content writing, brainstorming

Temperature 1.0+:
- Creative, diverse outputs
- Use for: Creative writing, idea generation
```

### 5. Handle Edge Cases Explicitly

```
Prompt:
"Classify customer feedback as positive, negative, or neutral.

Edge cases:
- If sarcastic, classify based on intent, not literal words
- If mixed sentiment, choose dominant sentiment
- If unclear, respond 'Unable to determine'

Feedback: [text]
Classification:"
```

### 6. Version Control Your Prompts

```python
# prompt_v1.txt
"Summarize: {text}"

# prompt_v2.txt
"Summarize in 3 sentences: {text}"

# prompt_v3.txt (current)
"""Summarize the following text in 3 sentences.
Focus on main ideas and key takeaways.
Text: {text}
Summary:"""

# Track which version works best
```

## Key Takeaways

### Fundamentals

1. **Prompt engineering is iterative** - test and refine

2. **Be specific** - clear instructions yield better results

3. **Provide context** - help the model understand the task

4. **Show examples** - few-shot learning improves performance

5. **Structure outputs** - specify format for consistent results

### Techniques

6. **Zero-shot** works for simple, common tasks

7. **Few-shot** improves performance on complex/specific tasks

8. **Chain-of-thought** enables better reasoning with "step by step"

9. **ReAct** combines reasoning with tool use

10. **Temperature** controls creativity vs consistency

### Best Practices

11. **Test edge cases** to ensure robustness

12. **Use delimiters** to separate prompt sections

13. **Iterate on prompts** based on outputs

14. **Version control** prompts like code

15. **Evaluate systematically** with metrics
